# 飞书上下文

每天自动缓存群消息与联系人。需要时读取缓存，不足则实时补充，按话题整合后呈现。

所有缓存为 markdown，直接读写。

## 缓存结构

```
memory/feishu-context/
├── groups.md          # 关注的群组
├── contacts.md        # 联系人
├── summary.md         # 活跃群与近期要点
└── daily/
    └── YYYY-MM-DD.md  # 每日消息快照（保留 7 天）
```

格式约定见 `references/formats.md`。

## 初始化

首次触发时，若 `memory/feishu-context/groups.md` 不存在：

1. 创建目录结构
2. 通过 `aily-im list-chats` 获取活跃群组，写入 `groups.md`
3. 创建空的 `contacts.md` 和 `summary.md`
4. 执行一次每日更新，生成首份快照
5. 创建定时任务：每天凌晨 1:00 执行每日更新

## 使用

收到与飞书上下文相关的请求时：

1. **读缓存** — 先读 `summary.md` 掌握全貌，再按需读 `daily/*.md`、`contacts.md`
2. **缓存够用** → 直接回答
3. **缓存不足** → 通过飞书 CLI 补充查询，新发现的群、联系人、消息同步写入缓存
4. **整合呈现** — 将相关人物、群组、文档、动态整合后回复用户

## 每日更新

定时任务自动执行：

1. 按 `groups.md` 逐群拉取当天消息（`aily-im list-messages`），写入 `daily/YYYY-MM-DD.md`
2. 更新 `summary.md`：每个活跃群一句话概括当天讨论重点
3. 清理 7 天前的快照

## 典型场景

| 用户说 | 关键动作 |
|--------|----------|
| "下午跟A聊B项目，帮我看看之前讨论了什么" | 定位A相关的群和消息，不足则搜索补充 |
| "A项目最近什么进展" | 读近几天 daily 快照，找相关讨论 |
| "帮我捋一下A方向相关的所有信息" | 跨多天检索，必要时通过 CLI 扩大范围 |
