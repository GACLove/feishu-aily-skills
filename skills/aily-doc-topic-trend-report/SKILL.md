# SKILL: aily-doc-topic-trend-report

根据用户指定的**主题**和**时间范围**，自动检索权威动态，生成一份**飞书云文档动态报告**，按类别聚类，每条新闻含 intro / description / 索引链接。强调**时间筛选 + 权威优先 + 去重**。

---

## 1) Overview

### ⚠️ 输出形式（强制）
- **本 skill 被触发即代表用户期望生成飞书云文档**，主 Agent 无需询问用户是否需要写成文档，也不得以对话文本替代文档作为最终交付物。
- 数据收集完成后，必须通过 `task`（subagent_type: writer）委托 Writer Agent 创建飞书云文档。

### 飞书工具优先（必须遵守）
- 所有飞书相关信息获取与写作，先参考 `feishu-use-skills-map` 并按其指引调用对应技能
- 需要访问飞书文档/群消息/会议/任务时，必须先 `get_skills` 获取对应飞书技能说明

### 工具/Skill 明确清单（执行前先 get_skills）

| Agent | 工具名 | 用途 | 使用阶段 |
|-------|--------|------|---------|
| 主 Agent | `web_search` | 多源检索公开信息 + 时间过滤 | Phase 2: 多源检索 |
| 主 Agent | `fetch` | 获取重要页面详情（提取时间、标题、核心内容） | Phase 2: 多源检索 / Phase 3: 时间核验 |
| 主 Agent | `todo_write` | 分解并追踪任务进度 | 全流程 |
| 主 Agent | `file` | 将整理后的素材写入 draft.md | Phase 4: 去重与分类 |
| 主 Agent | `task` | 分发写作任务给 Writer Agent | Phase 5: 生成报告 |
| Writer Agent | `file_read` | 读取 draft.md 素材 | 报告生成 |
| Writer Agent | `outline_generator` | 生成结构化大纲 | 报告生成 |
| Writer Agent | `feishu_doc_create` | 生成飞书云文档 | 报告生成 |

### 证据与时间约束（与主/写作agent一致）
- 任何数据/结论必须来自可追溯来源；不确定则明确标注"需补充资料"。
- 若用户指定时间范围，必须严格筛选在时间窗内的资料，过期内容不得使用。
- 不可编造事实、链接或来源；引用必须有明确出处。

### 可读性规则（强制）
- 并列项>=3：优先使用无序/有序列表或表格
- 单段>150字必须拆分（列表/小节/Callout）
- 连续两段无列表且信息密度高：插入列表或 Callout

### 触发方式（Triggers）
- "给我 {主题} 的日报/周报"
- "帮我整理 {日期/时间段} 的 {主题} 动态"
- "从 {日期} 到 {日期} 的 {主题} 新闻"
- "最近一周的 {主题} 动态"
- "{年月} 的 {主题} 大事"

### 输入（Inputs）

| 参数 | 是否必选 | 说明 | 示例 |
|------|---------|------|------|
| **主题** | 必选 | 报告聚焦的领域/关键词 | {主题关键词} |
| **时间范围** | 可选 | 支持多种格式（默认：今日） | 见下方时间格式说明 |
| **关注子领域** | 可选 | 主题下的细分方向 | {子领域1}、{子领域2}、{子领域3} |
| **关注公司/机构** | 可选 | 重点追踪的组织 | {公司1}、{公司2}、{公司3} |

#### 时间范围格式支持

| 格式类型 | 示例 | 解析结果 |
|---------|------|---------|
| 单日 | "今天"、"昨天"、"YYYY-MM-DD" | 当日 00:00-23:59 |
| 日期范围 | "M月D日到M月D日"、"YYYY-MM-DD ~ YYYY-MM-DD" | 起始日 00:00 ~ 结束日 23:59 |
| 相对时间 | "最近N天"、"过去一周"、"本月" | 自动计算时间窗口 |
| 年月 | "YYYY年M月"、"上个月" | 整月范围 |

### 核心输出（Deliverables）
- **飞书云文档主题动态报告**：按类别聚类、结构清晰、可追溯

### 质量标准
- **时间准确**：所有条目必须验证发布时间在指定窗口内
- **权威优先**：官方源 > 权威媒体 > 社区讨论
- **去重**：同一事件多来源只保留主条目，其他作为补充链接

## 3) Module A — 参数解析与多源检索（主 Agent 执行）

### Phase 1: 参数解析与确认

```
用户输入 → 解析主题 + 时间范围 → 确认参数
```

**Agent 行为**：
1. 从用户输入中提取：主题关键词、时间表述、可选的子领域/公司
2. 将时间表述转换为精确的时间窗口（UTC+8）
3. 向用户确认理解是否正确（如有歧义）

**示例**：
```
用户: "帮我整理上周的 {主题} 动态"
Agent 理解:
- 主题: {主题关键词}
- 时间窗口: {start_date} 00:00 ~ {end_date} 23:59 (UTC+8)
- 关注方向: {子领域1}、{子领域2}
```

---

### Phase 2: 多源检索

> **核心原则**：根据主题选择合适的信息源 + 时间过滤。**权威优先**。

#### 2.1 信息源选择（按主题动态调整）

| 主题类型 | 优先信息源 | 补充信息源 |
|---------|-----------|-----------|
| AI/科技 | 官方博客、arXiv、GitHub | Hacker News、Product Hunt、Twitter |
| 区块链/Web3 | 项目官网、CoinDesk、The Block | Twitter、Discord、GitHub |
| 游戏 | 官方公告、IGN、Polygon | Steam、Reddit、YouTube |
| 金融 | 官方公告、Reuters、Bloomberg | SEC Filings、财报 |
| 通用 | Google News、权威媒体 | 社交媒体、行业论坛 |

> 主题不在上表中时，参考"通用"行，优先选择该领域的官方源和头部媒体。

#### 2.2 检索策略

**web_search 查询模板**：
```
"{主题关键词}" (发布 OR 更新 OR 公告) {时间限定词}
site:{权威域名} "{主题关键词}" {日期}
"{关注公司}" "{主题关键词}" (release OR announcement) {时间}
```

**fetch 深入获取**：
```
fetch(
  url: "{搜索结果中的重要链接}",
  prompt: "提取发布时间、标题、核心内容、关键数据"
)
```

**写入 draft.md**：
每完成一批检索后，使用 `file` 追加写入素材：
```
file(path: "/home/workspace/draft.md", mode: "append")
```

## 4) Module B — 时间核验与分类（主 Agent 执行）

### Phase 3: 时间核验与筛选

> **本 Skill 的核心差异点**：严格的时间筛选是动态报告质量的生命线。

**核验规则**：
- 每条候选**必须**确认发布时间
- 时间不在指定窗口：**直接剔除**
- 无法确认时间：**不纳入**

**核验方式**（按优先级）：
1. 页面内查找发布时间/更新时间
2. URL 中的日期信息
3. 页面 meta 信息（`og:published_time` 等）

**fetch 核验示例**：
```
fetch(
  url: "{候选条目URL}",
  prompt: "确认该页面的发布时间/更新时间，精确到日期"
)
```

---

### Phase 4: 去重与分类

**去重规则**：
- 同一事件多来源：保留**原始来源**（官方 > 权威媒体 > 转载），其他作为补充链接
- 同一主体多更新：合并为一条，按时间排列要点

**通用分类框架**（按主题调整）：

| 类别编号 | 类别名称 | 适用内容 |
|---------|---------|---------|
| 一 | 官方发布 & 重大更新 | 来自官方的第一手信息 |
| 二 | 产品 & 技术 | 新产品、技术突破、版本更新 |
| 三 | 研究 & 论文 | 学术成果、研究报告 |
| 四 | 开源 & 工具 | 开源项目、开发者工具 |
| 五 | 行业 & 市场 | 投融资、合作、市场动态 |
| 六 | 观点 & 分析 | 深度分析、行业评论 |

> 仅保留有内容的类别，空类别不出现在最终报告中。

**整理后写入 draft.md**：
```
file(path: "/home/workspace/draft.md", mode: "append")
```

写入内容格式：
```markdown
## {类别名称}

### [{来源}] {标题}
- **发布时间**: {YYYY-MM-DD}
- **简介**: {1-2句概述}
- **详情**: {核心内容、关键数据、影响/意义}
- **链接**: {原文URL}
```

## 5) Module C — 调用 Writer Agent（主 Agent 执行）

### 任务分发

主 Agent 完成素材收集和分类后，调用 `task` 工具分发给 Writer Agent：

```json
{
  "description": "生成主题动态报告云文档",
  "prompt": "基于 /home/workspace/draft.md 中的主题动态素材，生成一份飞书云文档动态报告。\n\n要求：\n1. 严格按照动态报告模板结构\n2. 核心要点使用 Callout 突出\n3. 每条新闻含来源链接和发布时间\n4. 所有内容已通过时间核验\n5. 空类别不出现\n\n报告模板见 skill 中的文档结构模板。",
  "subagent_type": "writer"
}
```

### Writer Agent 执行流程

1. `file_read`: 读取 `/home/workspace/draft.md`
2. `outline_generator`: 按模板生成结构化大纲
3. `feishu_doc_create`: 生成飞书云文档
4. `end`: 结束

## 9) 常见问题处理

| 问题 | 处理方式 |
|------|---------|
| 时间范围模糊 | 向用户确认具体日期，转换为精确时间窗口 |
| 主题过于宽泛 | 建议用户缩小范围或指定子领域 |
| 某时间段无内容 | 如实告知，建议扩大时间范围 |
| 信息源受限 | 说明限制，使用替代源，在附录中注明 |
| 内容过多 | 按重要性筛选，保留 Top N，权威源优先 |
| 发布时间无法确认 | 该条目不纳入报告，在核验统计中计入剔除数 |
| 同一事件多来源冲突 | 保留原始来源为主条目，差异部分在详情中说明 |
