# SKILL: aily-doc-business-review-report

季度/年度业务汇报：围绕**指标达成、业务进展、问题与对策**输出结构化业务汇报。从多维表格、内部文档、日历、会议纪要等飞书生态数据源中获取业务数据，经过结构化分析后，生成一份可直接用于管理层汇报的飞书云文档。

---

## 1) Overview

### ⚠️ 输出形式（强制）
- **本 skill 被触发即代表用户期望生成飞书云文档**，主 Agent 无需询问用户是否需要写成文档，也不得以对话文本替代文档作为最终交付物。
- 数据收集完成后，必须通过 `task`（subagent_type: writer）委托 Writer Agent 创建飞书云文档。

### 飞书工具优先（必须遵守）
- 所有飞书相关信息获取与写作，先参考 `feishu-use-skills-map` 并按其指引调用对应技能
- 需要访问飞书文档/群消息/会议/任务时，必须先 `get_skills` 获取对应飞书技能说明
- 严禁跳过飞书工具直接使用外部搜索替代内部数据

### 工具/Skill 明确清单（执行前先 get_skills）
- 必须先参考 `feishu-use-skills-map` 并调用对应技能
- 主要技能与用途：

| 工具/技能 | 用途 | 适用场景 |
|-----------|------|---------|
| `aily-base` | 多维表格数据读取 | 获取 KPI 指标表、OKR 进度表、项目看板等结构化数据 |
| `aily-doc` | 飞书云文档搜索与读取 | 搜索项目文档、方案文档、历史汇报 |
| `knowledge_answer` | 内部知识库检索 | 查询 OKR 定义、KPI 口径说明、业务规范 |
| `aily-calendar` | 日历/日程读取 | 获取关键里程碑会议、评审会、复盘会日程 |
| `minute`（MCP） | 会议妙记读取 | 提取会议中的决策事项、行动项、进展汇报 |
| `file` | 将素材写入 draft.md（append 模式） | 每个数据步骤完成后 |

### 证据与时间约束（与主/写作 Agent 一致）
- 任何数据/结论必须来自可追溯来源；不确定则明确标注"需补充资料"
- 若用户指定时间范围（如 2025 Q1），必须严格筛选在时间窗内的资料，过期内容不得使用
- 不可编造事实、链接或来源；引用必须有明确出处
- 指标数据必须注明数据截止日期与数据来源表名

### 可读性规则（强制）
- 并列项>=3：优先使用无序/有序列表或表格
- 单段>150字必须拆分（列表/小节/Callout）
- 连续两段无列表且信息密度高：插入列表或 Callout
- 指标数据一律使用表格呈现，禁止纯文字罗列

### 触发方式（Triggers）
- "帮我写一份季度业务汇报"
- "生成 Q1 业务复盘报告"
- "做一份年度经营汇报"
- "把这个季度的 KPI 完成情况整理成汇报"
- "帮我准备季度经营分析报告"
- "年度业务总结"
- "经营复盘"

### 输入（Inputs）
- **必选**：
  - 业务口径（产品线/业务线/部门名称，如"国际电商业务"）
  - 指标口径（核心 KPI 名称列表，如 GMV、DAU、转化率、客单价）
  - 时间范围（如 2025 Q1、2024 年度）
- **可选**：
  - 关键项目清单（用户指定需要汇报的重点项目）
  - 目标值来源（OKR 表链接、KPI 看板链接）
  - 对比基准（同比/环比/对标竞品）
  - 汇报受众（CEO/VP/总监，影响详略程度）

### 核心输出（Deliverables）
- **飞书云文档业务汇报**：结构清晰、数据明确、可直接用于管理层汇报

### 质量标准
- **口径一致**：全文指标口径统一，与数据源定义一致
- **事实可信**：关键指标可追溯到多维表格或文档来源
- **结论有据**：每条分析结论必须有数据支撑，不做无依据推断
- **行动导向**：问题必须配对策，对策必须有负责人和时间节点
- **时效正确**：数据截止日期准确标注，不混用不同时间口径数据

## 3) Module A — 数据获取（主 Agent 执行）

> **核心原则**：优先使用飞书工具获取内部数据，所有数据写入 `draft.md` 并标注来源。每个数据获取步骤完成后立即写入 `draft.md`，确保素材不丢失。

### A1. 业务指标数据

#### 步骤一：从多维表格获取 KPI 数据

**工具**：`aily-base`

**操作方式**：
```
aily-base(
  action: "read_records",
  table: "业务KPI看板",
  filters: {
    time_range: "2025-01-01 ~ 2025-03-31",
    business_line: "国际电商业务"
  },
  fields: ["指标名称", "目标值", "实际值", "达成率", "同比", "环比", "数据截止日期"]
)
```

**获取内容**：
- 各 KPI 指标的目标值与实际值
- 达成率计算结果
- 同比/环比变化率
- 月度/周度趋势数据

#### 步骤二：从知识库获取 OKR/KPI 口径定义

**工具**：`knowledge_answer`

**查询策略**：
```
knowledge_answer(
  query_list: [
    "国际电商业务 2025 Q1 OKR 目标值是多少",
    "国际电商业务 GMV 指标口径定义",
    "国际电商业务 DAU 计算方式与统计规则",
    "国际电商业务 Q1 KPI 考核标准"
  ],
  explanation: "获取业务 OKR 目标值与指标口径定义，确保汇报口径一致"
)
```

## 4) Module B — 结构化分析（主 Agent 执行）

> **核心原则**：基于 Module A 获取的数据素材，进行结构化分析，形成判断与洞察，而非简单数据罗列。分析结果追加写入 `draft.md`。

### B1. 指标达成分析

**分析方法**：
- 逐项 KPI 对比实际值与目标值，计算达成率
- 对未达标指标进行差距量化与归因分析
- 对超额完成指标提炼成功因素

**写入 draft.md**（使用 `file(path: "/home/workspace/draft.md", mode: "append")`）：
```markdown
## B1. 指标达成分析

### 达标情况分类

#### 超额完成（达成率 >= 100%）
| 指标 | 目标值 | 实际值 | 达成率 | 成功归因 |
|------|--------|--------|--------|---------|
| DAU | 1200 万 | 1350 万 | 112.5% | 春节运营活动拉新效果超预期，推荐 V2 上线后用户访问频次提升 |
| 新客占比 | 30% | 35% | 116.7% | 东南亚新市场开拓带来大量新增用户 |

#### 未达标（达成率 < 100%）
| 指标 | 目标值 | 实际值 | 达成率 | 差距 | 根因分析 |
|------|--------|--------|--------|------|---------|
| GMV | 5.0 亿美元 | 4.6 亿美元 | 92.0% | -0.4 亿美元 | 客单价下滑与转化率不足共同导致，虽然流量增长超预期但未能有效转化为 GMV |
| 订单转化率 | 3.5% | 3.2% | 91.4% | -0.3pp | 2月改版后商品详情页加载速度下降，导致跳出率上升；部分品类缺货率较高 |
| 客单价 | 42 美元 | 39 美元 | 92.9% | -3 美元 | 新客占比提升但新客客单价偏低（平均 28 美元）；促销活动折扣力度大拉低均价 |
| 用户留存率 | 45% | 43% | 95.6% | -2pp | 新客留存率（32%）显著低于老客（56%），新客占比提升稀释整体留存 |
```

## 9) 常见失败模式与修复

| 失败模式 | 原因 | 修复方法 |
|---------|------|---------|
| 指标口径不一致 | 多维表格与 OKR 文档中的指标定义不同 | 在 A1 步骤中用 knowledge_answer 确认口径定义，全文统一 |
| KPI 数据缺失 | aily-base 多维表格权限不足或表名错误 | 向用户确认表名/链接，尝试 knowledge_answer 兜底获取 |
| 目标值缺失 | OKR 目标尚未录入系统或知识库中无记录 | 向用户确认目标值，或标注"目标值待确认"并在报告中留白 |
| 项目进展信息不全 | aily-doc 搜索结果不包含所有项目 | 用户提供关键项目清单，逐一搜索；结合 minute 会议纪要补充 |
| 会议纪要内容空泛 | 妙记未记录决策事项或行动项 | 结合 aily-calendar 会议日程推断会议主题，用 knowledge_answer 补充 |
| 分析停留在数据罗列 | 只堆砌数据未做归因分析 | 每个未达标指标必须执行根因拆解（内因/外因/贡献度量化） |
| 对策无法落地 | 对策过于笼统，缺少具体行动 | 每条对策必须包含：负责人、措施清单、时间节点、验收标准 |
| 同比环比计算错误 | 基期数据取错或公式有误 | 所有计算数据来自多维表格原始字段，不手动计算 |
| 报告结构混乱 | Writer Agent 未严格遵循模板 | 在 task prompt 中明确要求遵循模板，outline_generator 强制对齐 |
| 数据来源未标注 | draft.md 中缺少来源信息 | 每个数据获取步骤强制写入来源标注，Writer Agent 保留不删除 |
