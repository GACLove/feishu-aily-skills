# SKILL: aily-doc-industry-research

面向特定行业/赛道的**系统性深度研究**能力。从行业定义、市场规模、竞争格局、技术趋势、政策环境、投融资动态等多维度进行**广覆盖、深挖掘**的全面调研，输出结构化的行业研究报告。

---

## 1) Overview

### ⚠️ 输出形式（强制）
- **本 skill 被触发即代表用户期望生成飞书云文档**，主 Agent 无需询问用户是否需要写成文档，也不得以对话文本替代文档作为最终交付物。
- 数据收集完成后，必须通过 `task`（subagent_type: writer）委托 Writer Agent 创建飞书云文档。

### 飞书工具优先（必须遵守）
- 所有飞书相关信息获取与写作，先参考 `feishu-use-skills-map` 并按其指引调用对应技能
- 需要访问飞书文档/群消息/会议/任务时，必须先 `get_skills` 获取对应飞书技能说明

### 工具/Skill 明确清单（执行前先 get_skills）

| Agent | 工具名 | 用途 | 使用阶段 |
|-------|--------|------|----------|
| 主 Agent | `web_search` | 广泛检索公开信息（市场规模、新闻、报告） | Module A 广度检索 |
| 主 Agent | `fetch` | 获取特定网页/文档全文（报告、招股书、白皮书） | Module A/B 广度+深度 |
| 主 Agent | `aily-browser` | 交互式网页浏览（动态页面、需操作的数据源） | Module A/B 广度+深度 |
| 主 Agent | `knowledge_answer` | 检索内部知识库（公司内部信息、历史研究） | Module B 深度挖掘 |
| 主 Agent | `aily-doc` | 检索/操作飞书云文档（需先 `get_skills` 获取技能说明） | Module B 深度挖掘 |
| 主 Agent | `todo_write` | 任务分解与进度管理 | 全流程 |
| 主 Agent | `file` | 将素材写入 draft.md（`path: "/home/workspace/draft.md", mode: "append"`） | Module A/B/C |
| 主 Agent | `task` | 分发写作任务给 Writer Agent | Module D |
| Writer Agent | `file_read` | 读取 draft.md 素材 | 报告生成 |

| Writer Agent | `feishu_doc_create` | 创建飞书云文档 | 报告生成 |

### 证据与时间约束（与主/写作 Agent 一致）
- 任何数据/结论必须来自可追溯来源；不确定则明确标注"需补充资料"。
- 若用户指定时间范围，必须严格筛选在时间窗内的资料，过期内容不得使用。
- 不可编造事实、链接或来源；引用必须有明确出处。

### 可读性规则（强制）
- 并列项 >= 3：优先使用无序/有序列表或表格
- 单段 > 150 字必须拆分（列表/小节/Callout）
- 连续两段无列表且信息密度高：插入列表或 Callout

### 触发方式（Triggers）
- "帮我调研一下 {行业名} 行业/赛道"
- "做一份 {行业名} 领域的行业研究报告"
- "分析 {行业名} 行业的市场规模和竞争格局"
- "{行业名} 行业有哪些主要玩家？各自优劣势是什么？"
- "{行业名} 赛道的投融资情况和发展趋势"
- "我想进入 {行业名} 市场，帮我做个行业分析"

### 输入（Inputs）
- **必选**：行业/赛道名称或关键词
- **可选**：
  - 地理范围（全球/中国/北美/欧洲等）
  - 时间范围（默认：近 1-3 年）
  - 研究深度（概览/标准/深度）
  - 关注重点（市场规模/竞争格局/技术趋势/政策/投融资）
  - 对标公司（指定分析某些公司）

### 核心输出（Deliverables）
- **飞书云文档行业研究报告**：结构标准、数据详实、可追溯

### 研究原则
- **广度优先**：多维度、多来源、多视角全面覆盖
- **深度挖掘**：核心问题深入分析，不浮于表面
- **数据可信**：核心数据必须标注来源
- **时效性**：优先最新数据，旧数据标注年份

### 质量标准
- 核心数据至少 2 个独立来源交叉验证
- 所有数据格式：`数据（来源，年份）`
- 差异超过 20% 的数据需注明差异原因
- Top 3-5 公司必须深度分析，不可仅列名称
- 分析框架（五力/SWOT）必须完成，不可跳过

## 6) Module D — 调用 Writer Agent 生成报告

### D1. 任务分发

主 Agent 完成数据收集和分析后，调用 `task` 工具分发给 Writer Agent：

```json
{
  "description": "生成{行业名}行业研究报告云文档",
  "prompt": "基于 /home/workspace/draft.md 中的行业研究素材，生成一份结构完整的飞书云文档行业研究报告。\n\n要求：\n1. 严格按照行业研究报告模板结构\n2. Executive Summary 放在最前面，便于快速阅读\n3. 所有数据标注来源和年份\n4. 结论有数据支撑，避免主观臆断\n5. 报告篇幅：标准版 15-30 页\n\n报告模板见下方。",
  "subagent_type": "writer"
}
```

### D2. Writer Agent 执行流程

Writer Agent 收到任务后按以下步骤执行：

1. **`file_read`**：读取 `/home/workspace/draft.md` 全部素材
2. **`outline_generator`**：基于素材和报告模板生成大纲
3. **`feishu_doc_create`**：按大纲逐章节生成飞书云文档
4. **`end`**：结束任务，返回文档链接

- [ ] 投融资数据已获取

- [ ] 至少 2 份权威报告已 fetch 全文阅读

- [ ] 五力分析已完成

- [ ] 趋势研判有依据
- [ ] 机会与风险已总结

- [ ] Executive Summary 完整且精炼

- [ ] 核心数据有来源标注
- [ ] 结论有数据支撑

## 10) 常见失败模式与修复

| 失败模式 | 原因 | 修复方法 |
|---------|------|---------|
| 广度不足 | 检索维度不全 | 严格执行 6 大维度检索，逐维度完成 |
| 深度不够 | 只看摘要不读全文 | 用 fetch 获取核心报告全文 |
| 数据冲突 | 单一来源 | 交叉验证，差异需说明 |
| 堆砌无洞察 | 缺少分析框架 | 五力 + SWOT + 趋势研判缺一不可 |
| 来源不清 | 未标注 | 强制格式：`数据（来源，年份）` |
| 内部视角缺失 | 未查内部知识 | knowledge_answer + aily-doc 必须执行 |
| draft.md 为空 | 未写入草稿 | 每步完成后立即 `file(draft.md, append)` |
| Writer 输出混乱 | 未提供模板 | task prompt 中附带完整报告模板 |
