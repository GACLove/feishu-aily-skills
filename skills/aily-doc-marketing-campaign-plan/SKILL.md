# SKILL: aily-doc-marketing-campaign-plan

活动营销方案：明确目标人群、核心卖点、传播节奏、渠道矩阵、预算与风险，形成可执行方案。

---

## 1) Overview

### ⚠️ 输出形式（强制）
- **本 skill 被触发即代表用户期望生成飞书云文档**，主 Agent 无需询问用户是否需要写成文档，也不得以对话文本替代文档作为最终交付物。
- 数据收集完成后，必须通过 `task`（subagent_type: writer）委托 Writer Agent 创建飞书云文档。

### 飞书工具优先（必须遵守）
- 所有飞书相关信息获取与写作，先参考 `feishu-use-skills-map` 并按其指引调用对应技能
- 需要访问飞书文档/群消息/会议/任务时，必须先 `get_skills` 获取对应飞书技能说明

### 工具/Skill 明确清单（执行前先 get_skills）
- 必须先参考 `feishu-use-skills-map` 并调用对应技能
- 主要技能：`aily-doc`、`knowledge_answer`、`web_search`、`fetch`
- 内部历史活动/品牌资料：`aily-doc` / `knowledge_answer`
- 外部人群洞察/竞品案例/渠道数据：`web_search` + `fetch`
- draft 写入：`file`（将各步骤收集的素材 append 到 `/home/workspace/draft.md`）

### 证据与时间约束（与主/写作agent一致）
- 任何数据/结论必须来自可追溯来源；不确定则明确标注"需补充资料"。
- 若用户指定时间范围，必须严格筛选在时间窗内的资料，过期内容不得使用。
- 不可编造事实、链接或来源；引用必须有明确出处。
- 预算数字与 KPI 必须来自用户输入或可查证的行业基准，不可臆造。

### 可读性规则（强制）
- 并列项>=3：优先使用无序/有序列表或表格
- 单段>150字必须拆分（列表/小节/Callout）
- 连续两段无列表且信息密度高：插入列表或 Callout
- 时间线/节奏规划使用表格或分阶段小节，禁止纯文字堆砌

### 触发方式（Triggers）
- "营销方案"
- "活动策划"
- "推广计划"
- "帮我做一个活动营销方案"
- "XX 产品/品牌的推广方案"
- "策划一场 XX 活动"

### 输入（Inputs）
- **必选**：
  - 活动目标（品牌曝光/获客转化/新品首发/节日促销等）
  - 目标人群（人群画像或关键词描述）
- **可选**：
  - 预算范围（总预算或预算区间）
  - 活动时间周期（开始-结束日期）
  - 渠道偏好（线上/线下/全域；指定平台）
  - 品牌/产品名称
  - 竞品列表
  - 历史活动参考

### 核心输出（Deliverables）
- **飞书云文档活动营销方案**：结构清晰、策略可执行、预算可落地

### 质量标准
- **目标明确**：活动目标与 KPI 可量化、可追踪
- **人群精准**：画像有数据支撑，洞察有来源
- **策略可执行**：渠道、内容、节奏均有明确动作项与责任方
- **预算合理**：分配有依据，ROI 预估有基准参考
- **风险可控**：至少列出 3 项核心风险及对应预案

## 9) 常见失败模式与修复

| 失败模式 | 原因 | 修复方法 |
|---------|------|---------|
| 人群画像空洞 | 仅凭直觉描述，缺少数据支撑 | web_search 获取人群报告，用具体数据填充画像维度 |
| 传播主张无差异 | 未分析竞品，主张缺乏区分度 | 先完成竞品营销分析（A3），找到差异化空间再提炼主张 |
| 渠道选择拍脑袋 | 未参考渠道 ROI 基准与人群触媒数据 | 用 A4 的渠道效果数据 + A2 的触媒习惯交叉验证，选择最优渠道组合 |
| 节奏规划过于笼统 | 只写"预热-引爆-长尾"没有具体动作 | 每个阶段必须拆到具体日期 + 具体渠道 + 具体内容形式 + 负责人 |
| 预算分配无依据 | 凭经验拍数字，与 KPI 脱节 | 基于渠道 ROI 基准反推预算，确保"预算→KPI"可验证 |
| KPI 不可衡量 | 目标写"提升品牌知名度"无量化指标 | 所有 KPI 必须有数字目标 + 衡量方式 + 数据来源 |
| 风险预案缺失 | 方案过于乐观，无应急计划 | 强制至少列 3 项风险，每项含概率评估+影响程度+具体应对措施 |
| 内部资料未利用 | 跳过 aily-doc/knowledge_answer 直接外部检索 | 强制先执行 A1 内部检索，历史活动经验是最有价值的参考 |
| 数据来源缺失 | 报告中数据无出处 | 每个数据点必须标注"（来源，年份）"，无来源则标注"需补充资料" |
| 方案不可执行 | 策略写得宏大但无落地细节 | 执行排期必须含具体日期/负责人/产出物，渠道策略含内容形式/数量 |
