# SKILL: aily-doc-annual-report

文档创作年度报告：主要借助 `aily-doc` 获取用户年度文档信息（数量、点赞、评论、PV、UV 等），并基于这些指标与代表性文档产出年度总结型飞书云文档。

---

## 1) Overview

### ⚠️ 输出形式（强制）
- **本 skill 被触发即代表用户期望生成飞书云文档**，主 Agent 无需询问用户是否需要写成文档，也不得以对话文本替代文档作为最终交付物。
- 数据收集完成后，必须通过 `task`（subagent_type: writer）委托 Writer Agent 创建飞书云文档。

### 飞书工具优先（必须遵守）
- 所有飞书相关信息获取与写作，**先参考 `feishu-use-skills-map`** 并按其指引调用对应技能
- 需要访问飞书文档/群消息/会议/任务时，必须先 `get_skills` 获取对应飞书技能说明
- 禁止在有飞书原生工具可用的情况下使用 `web_search` 或 `fetch` 替代

### 工具/Skill 明确清单（执行前先 get_skills）

| 角色 | 工具/技能 | 用途 | 调用时机 |
|------|----------|------|---------|
| 主 Agent | `aily-doc` | 搜索/列出飞书云文档、获取文档详情与互动指标 | 数据获取阶段（Module A） |
| 主 Agent | `knowledge_answer` | 内部知识库补充，兜底获取遗漏信息 | 数据获取末尾（Module A4） |
| 主 Agent | `todo_write` | 任务分解与进度管理 | 初始化阶段 |
| 主 Agent | `file` | 将素材写入 draft.md（append 模式） | 每个数据获取子步骤完成后 |
| 主 Agent | `task` | 分发写作任务给 Writer Agent | 数据获取与分析全部完成后 |
| Writer Agent | `file_read` | 读取 draft.md 获取全部素材 | 写作阶段第一步 |
| Writer Agent | `outline_generator` | 基于素材生成结构化大纲 | 读取素材后 |
| Writer Agent | `feishu_doc_create` | 生成飞书云文档 | 大纲确认后 |
| Writer Agent | `end` | 结束 Writer Agent 执行流 | 云文档生成完成后 |

### 证据与时间约束（与主/写作 Agent 一致）
- 任何数据/结论必须来自可追溯来源；不确定则明确标注"需补充资料"
- 若用户指定时间范围，必须严格筛选在时间窗内的资料，过期内容不得使用
- 不可编造事实、链接或来源；引用必须有明确出处
- 年度范围默认为自然年（1月1日 00:00:00 至 12月31日 23:59:59），用户可自定义

### 可读性规则（强制）
- 并列项>=3：优先使用无序/有序列表或表格
- 单段>150字必须拆分（列表/小节/Callout）
- 连续两段无列表且信息密度高：插入列表或 Callout
- 数据指标使用表格展示，避免纯文字罗列

### 触发方式（Triggers）
- "帮我生成年度文档报告"
- "年度文档创作总结"
- "我今年写了哪些文档？"
- "统计一下我今年的文档创作情况"
- "生成 {YYYY} 年的文档年报"
- "帮我做个文档回顾"

### 输入（Inputs）

| 参数 | 是否必选 | 说明 | 示例 |
|------|---------|------|------|
| **年度范围** | 必选 | 报告统计的年度区间 | 2024、2024年、2024-01-01 至 2024-12-31 |
| **用户/作者** | 可选 | 文档所属作者（默认：当前用户） | 当前用户、@张三 |
| **部门/团队** | 可选 | 限定范围到某团队 | 产品团队、技术部 |
| **文档类型** | 可选 | 筛选特定类型文档 | 文档(docx)、表格(sheet)、幻灯片(slides) |
| **关注维度** | 可选 | 用户特别关注的分析方向 | 影响力、产出量、互动热度 |

### 核心输出（Deliverables）
- **飞书云文档年度报告**：以文档创作指标为核心的年度总结，包含数量统计、互动分析、Top 榜单、主题分布、趋势变化、亮点提炼

### 质量标准
- **指标完整**：年度文档总量、PV、UV、点赞、评论五大核心指标齐全
- **榜单清晰**：Top 文档按不同维度排列，附标题与链接
- **趋势可见**：按月/按季度展示产出与互动趋势
- **主题聚类**：文档按主题/类型分类统计
- **亮点突出**：年度代表性文档有定性描述，说明其影响力

**工具**：`aily-doc`（按文档逐条获取互动数据）

**调用方式**（对 A1 中每篇文档获取详情）：
```
aily-doc get-doc-detail(
  doc_token: "{文档token}",
  fields: ["pv", "uv", "likes", "comments", "shares"]
)
```

**获取内容**（每篇文档）：
- PV（页面浏览量）
- UV（独立访客数）
- 点赞数
- 评论数
- 分享次数（如可获取）

**处理策略**：
- 对所有文档逐一拉取互动指标
- 若文档数量过多（>50 篇），可按批次拉取，每批次 20 篇

- 汇总计算：年度总量、均值、中位数、最大值

### A3. 代表性文档抽样

**策略**：根据不同维度排序，选取各维度 Top N 文档作为年度亮点候选。

**排序维度与选取规则**：
1. **PV 排行 Top 5**：最受关注的文档
2. **UV 排行 Top 5**：触达范围最广的文档
3. **点赞排行 Top 5**：最受认可的文档
4. **评论排行 Top 5**：引发讨论最多的文档
5. **综合排行 Top 10**：多维度加权（PV 40% + UV 25% + 点赞 20% + 评论 15%）

**去重**：跨维度重复的文档只保留综合排名最高的一次。

**深度获取**（对 Top 10 综合排名文档）：
```
aily-doc get-doc-content(
  doc_token: "{文档token}",
  fields: ["title", "summary", "create_time", "update_time", "word_count"]
)
```

## 4) Module B — 结构化分析（主 Agent 执行）

> **核心原则**：基于 Module A 收集的原始数据，进行多维度分析，形成洞察与结论，全部写入 `draft.md`。

### B1. 年度概览统计

**分析内容**：
- 年度文档总产出量及同比变化（若有往年数据）

outline_generator(

feishu_doc_create(

- 返回飞书云文档链接

## 9) 常见失败模式与修复

| 失败模式 | 原因 | 修复方法 |
|---------|------|---------|
| 文档列表不完整 | aily-doc 单次返回数量限制 | 按季度分批拉取（每批 200 篇），合并去重 |
| 互动指标缺失 | 某些文档的 PV/UV 数据获取失败 | 标注"数据缺失"，在汇总时注明有效样本量，不影响整体分析 |
| 年度范围错误 | 时间过滤条件设置不正确 | 严格使用 YYYY-01-01 ~ YYYY-12-31 格式，确认返回数据的创建时间在范围内 |
| Top 榜单数据不一致 | 综合排名与单项排名使用了不同数据快照 | 所有榜单基于同一份互动指标数据（A2）计算，不重复拉取 |
| 主题分类不准确 | 仅依赖标题关键词，分类粗糙 | 结合文档摘要和标题进行聚类，必要时对边界案例人工复核 |
| 亮点提炼流于数字 | 只有指标排名，缺少定性描述 | 结合 knowledge_answer 补充的背景信息（获奖、被引用、会议推荐等）丰富描述 |
| draft.md 信息过载 | 写入内容冗余，Writer Agent 难以提取关键信息 | draft.md 中每个 section 末尾添加"洞察"小节，Writer Agent 优先使用洞察结论 |
| 文档结构不符模板 | Writer Agent 未严格按模板执行 | 在 task prompt 中明确要求遵循模板，outline_generator 步骤强制使用模板结构 |
| 知识库兜底返回空结果 | knowledge_answer 未找到相关信息 | 正常情况，不强求补充，在 draft.md 中标注"knowledge_answer 无补充信息" |
| 链接失效或格式错误 | 飞书文档链接格式变化 | 使用 aily-doc 返回的原始 URL，不手动拼接或修改链接格式 |
