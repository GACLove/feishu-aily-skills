# SKILL: aily-doc-competitive-analysis

竞品分析：通过统一维度对比产品与市场信息，形成结构化对比结论与机会点。针对目标产品及其竞品，从产品定位、功能体验、定价策略、市场表现、用户口碑、技术架构等多维度进行**对齐口径、深度对比**的全面分析，输出结构化的竞品分析报告。

---

## 1) Overview

### ⚠️ 输出形式（强制）
- **本 skill 被触发即代表用户期望生成飞书云文档**，主 Agent 无需询问用户是否需要写成文档，也不得以对话文本替代文档作为最终交付物。
- 数据收集完成后，必须通过 `task`（subagent_type: writer）委托 Writer Agent 创建飞书云文档。

### 飞书工具优先（必须遵守）
- 所有飞书相关信息获取与写作，先参考 `feishu-use-skills-map` 并按其指引调用对应技能
- 需要访问飞书文档/群消息/会议/任务时，必须先 `get_skills` 获取对应飞书技能说明

### 工具/Skill 明确清单（执行前先 get_skills）
- 必须先参考 `feishu-use-skills-map` 并调用对应技能
- 主要技能：`web_search`、`fetch`、`aily-doc`、`knowledge_answer`

| 工具 | 用途 | 适用场景 |
|------|------|---------|
| `web_search` | 广泛检索公开信息 | 产品功能、评测、定价、市场份额、用户评价 |
| `fetch` | 获取特定网页/文档全文 | 竞品官网、产品文档、定价页面、API 文档 |
| `aily-doc` | 检索飞书云文档 | 内部已有的竞品分析、产品对比文档 |
| `knowledge_answer` | 检索内部知识库 | 公司内部竞品情报、产品规划、战略分析 |
| `file` | 将素材写入 draft.md（append 模式） | 每个数据步骤完成后 |

### 证据与时间约束（与主/写作agent一致）
- 任何数据/结论必须来自可追溯来源；不确定则明确标注"需补充资料"。
- 若用户指定时间范围，必须严格筛选在时间窗内的资料，过期内容不得使用。
- 不可编造事实、链接或来源；引用必须有明确出处。
- 竞品数据必须标注获取日期，产品功能和定价可能随时变化。

### 可读性规则（强制）
- 并列项>=3：优先使用无序/有序列表或表格
- 单段>150字必须拆分（列表/小节/Callout）
- 连续两段无列表且信息密度高：插入列表或 Callout
- 竞品对比优先使用表格呈现，确保横向可比

### 触发方式（Triggers）
- "竞品分析"
- "对比 A 和 B"
- "竞对研究"
- "帮我分析一下 XX 的竞品"
- "XX 和 YY 有什么区别？各自优劣势？"
- "做一份 XX 产品的竞品调研"
- "XX 领域主要竞品对比"

### 输入（Inputs）
- **必选**：目标产品名称 + 竞品列表（至少 1 个竞品，建议 2-5 个）
- **可选**：
  - 对比维度（功能/价格/技术/体验/市场，默认全维度）
  - 产品版本范围（如"仅对比企业版"、"免费版 vs 付费版"）
  - 时间范围（默认：当前最新版本及近 1 年数据）
  - 关注重点（如"重点关注 AI 能力对比"、"侧重定价策略"）
  - 目标受众（如"给产品经理看"、"给管理层决策用"）

### 核心输出（Deliverables）
- **飞书云文档竞品分析报告**：对比清晰、维度统一、结论可追溯、建议可执行

### 质量标准
- **证据驱动**：每条结论必须有公开信息或内部数据支撑，不做主观臆断
- **口径一致**：所有竞品在同一维度下使用相同标准评估，不偏袒任何一方
- **时效性**：优先引用最新版本和最新数据，旧数据需标注年份
- **可操作**：最终建议必须具体可落地，而非泛泛而谈

## 3) Module A -- 信息收集（主 Agent 执行）

> **核心原则**：以统一维度为纲，多工具、多来源交叉验证，确保对比口径一致。每个竞品必须在相同维度下收集信息，不可遗漏。

### A1. 确定竞品范围与对比维度

**目标**：在正式检索前，先明确"比谁"和"比什么"，避免信息收集失焦。

**确定竞品范围**：
- 直接竞品：同品类、同目标用户的产品（如 Notion vs 飞书文档 vs 语雀）
- 间接竞品：满足相同需求但形态不同的产品（如 Notion vs Obsidian vs Confluence）
- 潜在竞品：可能进入该领域的跨界玩家（如大厂新业务线）

**确定对比维度**（默认 6 大维度，用户可自定义）：

| 维度 | 说明 | 关键指标 |
|------|------|---------|
| 产品定位 | 目标用户、核心价值主张、品牌调性 | slogan、官网首屏文案、目标客群描述 |
| 功能与体验 | 核心功能清单、差异化功能、交互体验 | 功能覆盖率、独有功能数、用户体验评分 |
| 定价与商业模式 | 价格体系、免费/付费策略、变现方式 | 客单价、免费版功能范围、付费转化路径 |
| 渠道与获客 | 分发渠道、获客方式、生态合作 | 渠道覆盖数、合作伙伴、应用市场排名 |
| 用户与市场 | 用户规模、市场份额、用户画像 | DAU/MAU、市占率、NPS 评分 |
| 技术与架构 | 技术栈、API 开放程度、数据安全 | 技术选型、API 数量、安全认证 |

**写入 draft.md**（使用 `file(path: "/home/workspace/draft.md", mode: "append")`）：
```markdown
## 竞品范围与维度

### 竞品选取
| 竞品 | 类型 | 选取理由 |
|------|------|---------|
| 飞书文档 | 目标产品 | 本次分析主体 |
| Notion | 直接竞品 | 全球知识管理工具标杆，功能体系最完整 |
| 语雀 | 直接竞品 | 国内主要竞品，蚂蚁集团旗下，开发者群体渗透率高 |
| Confluence | 直接竞品 | 企业级协作文档老牌产品，Atlassian 生态核心 |
| Obsidian | 间接竞品 | 本地优先的知识管理工具，插件生态活跃 |

### 对比维度
本次分析覆盖以下 6 个维度：产品定位、功能与体验、定价与商业模式、渠道与获客、用户与市场、技术与架构。

> 来源：根据用户需求确定，YYYY-MM-DD
```

### A2. 产品功能与体验

**目标**：全面收集每个竞品的功能清单、差异化特性、用户体验评价。

```
- "{产品A} 功能 特性 更新日志 2025"
- "{产品A} 核心功能 功能列表 产品介绍"
- "{产品A} vs {产品B} 对比 评测"
- "{产品A} 新功能 产品更新 changelog"
- "{产品A} 用户体验 交互设计 UI 评价"
```

**fetch 获取产品官网详情**：
```
fetch(
```

## 9) 常见失败模式与修复

| 失败模式 | 典型表现 | 根因分析 | 修复方法 |
|---------|---------|---------|---------|
| 对比口径不一致 | A 产品列出 10 个功能、B 产品只列出 3 个；用不同颗粒度评估不同产品 | 未提前统一对比维度和功能粒度 | A1 阶段必须先确定维度框架，所有竞品在同一功能清单下逐项比对 |
| 信息时效性混乱 | 用 2023 年的定价数据与 2025 年的功能列表混合比较 | 未标注数据年份，未验证时效性 | 所有数据强制标注获取日期，定价和功能信息必须 fetch 最新官网 |
| 只列事实无分析 | 报告全是信息罗列，没有 SWOT、没有机会点、没有行动建议 | 跳过 Module B 结构化分析直接交付写作 | 严格执行 Module B 全部分析步骤，确保 draft.md 包含分析结论 |
