# SKILL: aily-doc-pre-meeting-brief

会前材料准备：围绕会议议题、决策点与材料清单，结合历史会议纪要与当前任务进展，生成可讨论、可决策的会前材料。

---

## 1) Overview

### ⚠️ 输出形式（强制）
- **本 skill 被触发即代表用户期望生成飞书云文档**，主 Agent 无需询问用户是否需要写成文档，也不得以对话文本替代文档作为最终交付物。
- 数据收集完成后，必须通过 `task`（subagent_type: writer）委托 Writer Agent 创建飞书云文档。

### 飞书工具优先（必须遵守）
- 所有飞书相关信息获取与写作，先参考 `feishu-use-skills-map` 并按其指引调用对应技能
- 需要访问飞书文档/群消息/会议/任务时，必须先 `get_skills` 获取对应飞书技能说明
- 严禁跳过飞书工具直接使用外部搜索替代内部数据

### 工具/Skill 明确清单（执行前先 get_skills）
- 必须先参考 `feishu-use-skills-map` 并调用对应技能
- 主要技能：`aily-im`、`aily-doc`、`aily-calendar`、`aily-user`、`aily_lark_task`(MCP)、`minute`(MCP)、`knowledge_answer`
- 会议/日程：`aily-calendar`（获取会议详情、参会人、会议描述、关联资源）
- 历史妙记：`minute` MCP（获取上次同系列会议纪要、决策、待办）
- 待办任务：`aily_lark_task` MCP（检查 action items 完成状态）
- 群聊消息：`aily-im`（搜索项目群中与会议议题相关的讨论）
- 云文档：`aily-doc`（搜索相关技术方案/PRD/数据文档）
- 人员信息：`aily-user`（查询参会人信息、职责）
- 兜底补全：`knowledge_answer`（前述工具未覆盖时使用）
- draft 写入：`file`（将各步骤收集的素材 append 到 `/home/workspace/draft.md`）

### 证据与时间约束（与主/写作 Agent 一致）
- 任何数据/结论必须来自可追溯来源；不确定则明确标注"需补充资料"。
- 若用户指定时间范围，必须严格筛选在时间窗内的资料，过期内容不得使用。
- 不可编造事实、链接或来源；引用必须有明确出处。
- 上次会议纪要中的待办/决策/结论，必须标注原始来源（妙记链接或文档链接）。

### 可读性规则（强制）
- 并列项>=3：优先使用无序/有序列表或表格
- 单段>150字必须拆分（列表/小节/Callout）
- 连续两段无列表且信息密度高：插入列表或 Callout
- 待决事项必须用表格呈现备选方案对比，禁止纯文字堆砌

### 触发方式（Triggers）
- "准备会前材料"
- "会议预读材料"
- "会前 brief"
- "帮我准备 XX 会议的材料"
- "生成会前文档"
- "整理一下明天会议要用的资料"

### 输入（Inputs）
- **必选**：会议主题、会议时间、参会对象
- **可选**：会议目标、争议点/待决事项、上次会议链接、相关项目群名称

### 核心输出（Deliverables）
- **飞书云文档会前材料**：议题明确、证据充分、决策点清晰、材料清单齐全

### 质量标准
- **决策导向**：每个待决事项都有备选方案和对比分析，参会人阅读后可直接讨论和决策
- **准备充分**：上次会议结论完整回顾、待办完成情况逐项核实、需提前阅读的材料全部列出并附链接
- **结构清晰**：使用表格、Callout、列表等结构化手段，5分钟内可快速浏览核心要点
- **时效性**：所有引用数据为最新状态，标注数据获取时间

## 3) Module A — 资料获取与准备（主 Agent 执行）

> **核心原则**：优先使用飞书工具按步骤获取六类信息源，确保会前材料的每一个论点都有据可查。最后用 `knowledge_answer` 兜底补全。

### A1. 会议基本信息

**工具**：`aily-calendar`

**获取内容**：
- 会议标题、开始/结束时间、时长
- 参会人列表及角色（组织者/必选参会/可选参会）
- 会议描述/备注（通常包含议题或背景说明）
- 会议关联资源（共享文档、妙记等）
- 会议所属系列（周例会/双周会/专项评审等）

**调用示例**：
```
aily-calendar: 搜索会议
- 关键词: "{会议主题}"
- 时间范围: "{会议日期前后1天}"
- 获取: 会议详情、参会人、描述、关联资源
```

**分析维度**：
- 确认会议系列（用于查找上次同系列会议）
- 提取会议描述中的议题线索
- 识别关键参会人及其职责


## 9) 常见失败模式与修复

| 失败模式 | 原因 | 修复方法 |
|---------|------|---------|
| 找不到上次会议纪要 | 会议系列名称不一致或 minute MCP 搜索关键词不匹配 | 放宽搜索条件（使用会议主题关键词而非完整标题），扩大时间范围至 60 天；若仍无结果，用 `aily-im` 在群聊中搜索会议纪要 |
| 待办状态信息不全 | `aily_lark_task` 中任务描述与会议纪要中的行动项措辞不同 | 按负责人 + 时间范围批量查询，人工匹配行动项；必要时在 draft.md 中标注"状态未确认，需会前与负责人核实" |
| 群聊讨论噪声多 | 搜索关键词太宽泛，返回大量不相关消息 | 缩小时间范围（上次会议至今），使用更精确的关键词组合，按发送人过滤（仅保留参会人消息） |
| 关联文档遗漏 | 文档标题不含会议议题关键词 | 多轮搜索：先按议题关键词搜索，再按参会人搜索近期文档，最后用 `knowledge_answer` 补充 |
| 备选方案分析不充分 | draft.md 中缺少对比维度或数据支撑 | 每个待决事项必须列出至少 2 个方案，对比维度至少包含：效果、成本、风险、周期；缺少数据时标注"需补充" |
| 文档内容流水账 | Writer Agent 照搬 draft.md 原始数据，缺少提炼 | 在 prompt 中明确要求：核心要点 Callout 必须高度概括，每个章节有结论性语句 |
| 时间分配不合理 | 议题数量与会议时长不匹配 | 检查议题时间分配总和不超过会议时长的 90%（预留 10% buffer），若超出则建议拆分会议或精简议题 |
| 材料链接失效 | 文档已删除或权限变更 | 获取文档时记录标题和作者，链接失效时标注"需联系@作者确认文档状态" |
| 会前材料发送过晚 | 材料生成时间距会议开始不足 | 建议至少在会议前 24 小时生成会前材料，给参会人充足的阅读时间 |
